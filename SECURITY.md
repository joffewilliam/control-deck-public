# Security Policy

## Reporting a vulnerability

Please report security issues **privately** — do not open a public issue for a
suspected vulnerability.

Use GitHub's private vulnerability reporting:
**Security → Report a vulnerability** on this repository
(<https://github.com/joffewilliam/control-deck-public/security/advisories/new>).

Include what you found, how to reproduce it, and the impact. You'll get an
acknowledgement as soon as possible. Security reports are prioritized.

## Scope

This repository hosts the downloads, docs, and feedback for Control Deck.
Reports about the application itself (installer, updater, agent execution, secret
handling) are in scope.

## Please do not include secrets

When sharing logs or reproductions, never include API keys, OAuth tokens, license
keys, private prompts, or customer data.
